Concepts critical to success

Understanding how to optimally set up {{appname}} jobs and workflows is critical to success. Here is how you can use them in your pipeline.

Jobs

A job is a collection of steps that are executed in a virtual environment. Each job has two required components: steps and executors.

jobs:
  build:
    docker:                  # executor
      - image: foo:bar
    steps:                   # steps
      - run: echo Run some tests

You can set up optimizations to help speed up your jobs:

  • Dependency caching - reuse data from expensive fetch operations from previous jobs
  • Parallelism - split up your test suite across multiple environments to increase speed
  • Resource classes - optimize compute power for a given job
  • And more optimizations
Workflows

A workflow is a set of rules for defining a collection of jobs and their run order. Workflows allow you to define and orchestrate how the steps of job execution are run. Some best practices for using workflows:

  • Optimize for speed - run independent jobs concurrently to speed up build times
  • Manual approval jobs - pause a workflow and wait for approval
  • Branch-level execution - only run a given set of jobs on specific branches
